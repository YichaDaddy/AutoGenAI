本週重點在於完成 AI 加速器 (NPU) IP 的 RTL Freeze，以及解決 PCIe Gen5 模組在靜態時序分析 (STA) 上的違例問題。目前整體 Tape-out 時程進度在控，但 PCIe 模組需要 APR 團隊協助調整 Floorplan。

專案：NPU IP RTL 設計與整合 RTL Freeze 達標

已完成 MAC Array 核心運算邏輯的 Coding，並通過了初步的 Linting 與 CDC (跨時鐘域) 檢查。

▶ 點擊展開技術細節與下週計畫

專案：PCIe Gen5 PHY 系統整合 時序違例 (Timing Violation)

在 SSG corner (慢速條件) 下，發現 Controller 與 PHY 之間的 Wrapper layer 有嚴重的 Setup time violation。

⚠️ Blocker 警告： 最差時序路徑 (Worst Negative Slack) 達到 -0.15ns，主要發生在 Data Formatting 的組合邏輯層，可能會影響下個月的 Tape-out 時程。
▶ 點擊展開分析與解決方案 (Action Items)
  • 根本原因： Mux 選擇邏輯過於複雜，導致 Logic Depth 太深。
  • Action Item 1 (我方)： 下週二前完成 RTL Pipeline 拆分，插入一個 Register stage 來打斷 Critical path。
  • Action Item 2 (跨部門)： 已發信請 APR 團隊評估是否能將該 Block 的 Macro 擺放位置拉近，減少 Wire delay。
